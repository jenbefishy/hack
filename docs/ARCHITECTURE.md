# Архитектура проекта

Проект организован как конвейер обработки сигнала:

1. Загрузка IQ-данных из `data/` (`.cf32`, `.npy`, `.bin`).
2. FM-демодуляция.
3. Детекция HSYNC/VSYNC.
4. Реконструкция кадров по синхроимпульсам.
5. Сохранение результатов и графиков.

## Слои

- **`signal_processing.py`**  
  Низкоуровневые операции и метрики сигнала: сглаживание, оценка периодичности, энергия, загрузка входных данных.

- **`sync_detection.py`**  
  Поиск синхроимпульсов и интервалов VSYNC с набором эвристик, чтобы отсечь ложные HSYNC-подобные фрагменты.

- **`reconstruction.py`**  
  Восстановление кадров: поиск стабильных строк, адаптивная нарезка активной части строки, корреляционное выравнивание.

- **`visualization.py`**  
  Сохранение диагностических графиков и изображений кадров.

- **`pipeline.py`**  
  Оркестрация шагов и формирование JSON-отчётов.

- **`main.py`**  
  CLI-интерфейс для запуска пайплайна из командной строки.

## Артефакты

Выходная директория (`--out`) содержит:
- `.npy` массивы с демодулированным сигналом и синхроимпульсами;
- `.json` отчёты;
- `.png` диагностические изображения.
